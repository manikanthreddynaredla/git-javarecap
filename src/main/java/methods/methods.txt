# ‚úÖ **Types of Methods in Java**

In Java, methods can be classified in multiple ways:

---

# **1Ô∏è‚É£ Based on Return Type**

## **A. Methods With Return Type**

These methods return a value (int, boolean, String, object, etc.).

### ‚úî Example:

```java
int add(int a, int b) {
    return a + b;
}
```

### ‚úî Use Case:

* When computation result is needed later.
* Example: addition, validations, searching, fetching data.

---

## **B. Methods Without Return Type (void)**

These methods **do not return any value**.

### ‚úî Example:

```java
void printMessage() {
    System.out.println("Hello!");
}
```

### ‚úî Use Case:

* Logging, printing, updating UI.
* Performing actions instead of returning data.

---

# **2Ô∏è‚É£ Based on Parameters**

## **A. Methods With Parameters**

Accept values while calling.

### ‚úî Example:

```java
void greet(String name) {
    System.out.println("Hello " + name);
}
```

### ‚úî Use Case:

* To customize behavior based on input.
* Example: payment(amount), login(username, password)

---

## **B. Methods Without Parameters**

Do not accept any arguments.

### ‚úî Example:

```java
void displayDate() {
    System.out.println(LocalDate.now());
}
```

### ‚úî Use Case:

* When no external input is required.

---

# **3Ô∏è‚É£ Based on Method Type (Very Important for Interviews)**

# **3.1 Static Methods**

* Belong to the class, not objects.
* Can be called without object creation.
* Used for utility and helper operations.

### ‚úî Example:

```java
class MathUtil {
    static int square(int x) {
        return x * x;
    }
}
```

### ‚úî Use Case:

* Utility methods (Math.sqrt(), Arrays.sort())
* Common reusable functions

---

# **3.2 Instance Methods**

* Require object creation.
* Can access instance variables.

### ‚úî Example:

```java
class Employee {
    String name;

    void showName() {
        System.out.println(name);
    }
}
```

### ‚úî Use Case:

* When behavior depends on instance data.

---

# **3.3 Abstract Methods**

* Declared without implementation.
* Must be implemented in subclass.

### ‚úî Example:

```java
abstract class Animal {
    abstract void sound();
}
```

### ‚úî Use Case:

* Designing frameworks
* Forcing sub-classes to follow a contract

---

# **3.4 Final Methods**

* Cannot be overridden.

### ‚úî Example:

```java
class Vehicle {
    final void start() {
        System.out.println("Vehicle starting...");
    }
}
```

### ‚úî Use Case:

* To prevent changing critical logic.

---

# **3.5 Synchronized Methods**

* Used in multithreading to avoid race conditions.

### ‚úî Example:

```java
synchronized void increment() {
    count++;
}
```

### ‚úî Use Case:

* Bank transactions
* Shared resource update

---

# **3.6 Native Methods**

* Implemented in languages like C/C++ using JNI.

### ‚úî Example:

```java
public native void startProcess();
```

### ‚úî Use Case:

* Hardware-level operations
* High-performance modules (video processing)

---

# **3.7 Recursive Methods**

* A method calling itself.

### ‚úî Example:

```java
int factorial(int n) {
    if (n == 1) return 1;
    return n * factorial(n - 1);
}
```

### ‚úî Use Case:

* Tree traversal
* Mathematical computations

---

# **4Ô∏è‚É£ Based on Access Modifier**

| Modifier      | Accessible From  | Use Case           |
| ------------- | ---------------- | ------------------ |
| **public**    | everywhere       | API methods        |
| **private**   | within class     | encapsulation      |
| **protected** | subclass/package | inheritance        |
| **default**   | same package     | package-level APIs |

---

# **5Ô∏è‚É£ Special Methods**

## **5.1 Constructor (not actually a method, but often asked)**

Used to initialize objects.

### ‚úî Example:

```java
class Car {
    Car() {
        System.out.println("Car created");
    }
}
```

---

## **5.2 Getter & Setter Methods**

Used for encapsulation.

### ‚úî Example:

```java
class Student {
    private int age;

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}
```
